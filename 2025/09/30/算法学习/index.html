数据结构

单链表

单链表支持以下操作：

1.向链表头插入一个数

2.删除第k个插入的数后面的数

3.在第K个插入的数后面插入一个数

双链表

双链表支持以下操作：

  1. 在最左侧插入一个数；
  2. 在最右侧插入一个数；
  3. 将第 k 个插入的数删除；
  4. 在第 k 个插入的数左侧插入一个数；
  5. 在第 k 个插入的数右侧插入一个数

模拟栈

模拟栈支持以下操作：

  1. push x – 向栈顶插入一个数 x；
  2. pop – 从栈顶弹出一个数；
  3. empty – 判断栈是否为空；
  4. query – 查询栈顶元素

模拟队列

模拟队列支持以下操作：

  1. push x – 向队尾插入一个数 x；
  2. pop – 从队头弹出一个数；
  3. empty – 判断队列是否为空；
  4. query – 查询队头元素

单调栈

  1. 利用栈这种数据结构的性质来维护某种单调性，分为单调递增栈和单调递减栈
  2. 利用栈来维护一个单调递增（或递减）的序列，从而在 O(n) 时间内快速找到每个元素左边/右边的第一个比它大（或小）的元素
  3. 给定一个长度为N的整数数列，输出每个数左边第一个比它小的数，如果不存在则输出-1

01背包问题

状态方程 f[i][j] = max(f[i - 1][j], f[i - 1][j - v[i]] + w[i]) 可以优化成一维数组，第二重循环倒序

完全背包问题

f[i][j] = max(f[i - 1][j], f[i - 1][j - v[i]] + w[i], f[i - 1][j - 2 * v[i]] + 2 * w[i] ...)

f[i][j - v[i]] = max( f[i - 1][j - v[i]] + w[i], f[i - 1][j - 2 * v[i]] + 2 * w[i] ...)

后面多出来一项由于满足完全背包问题的条件因此可以省略

因此状态方程 f[i][j] = max(f[i - 1][j], f[i - 1][j - v[i]] + w[i])可以优化成一维数组，第二重循环正序

多重背包问题

状态方程f[i][j] = max(f[i - 1][j], f[i - 1][j - v[i]] + w[i], f[i - 1][j - 2 * v[i]] + 2 * w[i] ...)

对于简单数据可以直接三重循环遍历写成f[i][j] = max(f[i][j], f[i - 1][j - k * v[i]] + k * w[i]) 其中max函数中的第一项是f[i][j]，在k=0的时候已经是f[i - 1][j]

对于复杂数据要进行优化，主要的优化方法有二进制优化，滑动窗口优化和完全背包优化等

但完全背包优化有个条件 要求每个物品的数量大于背包数量 即 Si * Vi > V,对于一般题目我们无法判断其数据，因此一般采用其他两种方法

二进制优化法：把第 i 类物品的 $s_i$件拆成若干块，1，2，4，8，… 这样对于任何一个数都可以选取其中的若干块组合得到，

如14 = 1+2+4+7。每一块只能选或不选，选最多选一次。这样该问题就转换成01背包问题。

贪心算法

贪心算法没有一个固定的模板。

其主要思想是利用局部最优解求得全局最优解，一般是单调的。

对于一个题目如果找不出局部最优不是全局最优反例就可以试试贪心算法
